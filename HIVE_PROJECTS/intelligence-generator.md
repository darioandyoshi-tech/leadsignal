# Intelligence Brief Generator (OEW-v1)

**Purpose:** Convert raw vendor outage data into a high-value "Executive Intelligence Brief" for B2B lead generation.

## Template Structure

### 1. Header: The "Gap" Alert
- **Vendor:** [Vendor Name]
- **Status:** 🔴 CRITICAL / 🟡 DEGRADED
- **PulseWatch Detection:** [Timestamp]
- **Official Status Page:** [Timestamp or "Not Yet Updated"]
- **The Gap:** [Time difference in minutes] $\rightarrow$ *This is the value proposition.*

### 2. The "Blast Radius" (Impact Analysis)
- **Affected Services:** [Specific APIs/Regions affected]
- **Observed Behavior:** [e.g., 504 Gateway Timeouts, Latency > 5s]
- **Community Signal:** [Summary of X/GitHub/Reddit complaints]
- **Confidence Level:** [e.g., 98% - Verified across 3 regions]

### 3. Actionable Intelligence (The "Save")
- **Immediate Mitigation:** [e.g., "Switch to backup region US-West-2", "Disable webhook retries"]
- **Risk Assessment:** [High/Med/Low] impact on customer-facing checkout/login.

### 4. The Offer
- **The Source:** "This intelligence was generated by the HIVE/PulseWatch autonomous monitoring layer."
- **Call to Action:** "Get this 60-minute head-start on every vendor outage. [Subscription Link/Contact]"

---

## Generation Logic (The Prompt)

When an incident is detected, the agent will:
1. Fetch current status from PulseWatch MCP.
2. Scrape latest 10 mentions on X/GitHub for the vendor.
3. Map findings to the template above.
4. Output in Markdown for professional delivery.
